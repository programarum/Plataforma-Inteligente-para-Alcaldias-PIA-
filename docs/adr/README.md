# Architecture Decision Records

Los Architecture Decision Records (ADR) conservan el contexto, la decision y las consecuencias de elecciones tecnicas relevantes para PIA.

## Estados

- `Proposed`: en discusion y sin autorizacion de implementacion.
- `Accepted`: decision vigente.
- `Deprecated`: decision conservada como referencia, pero desaconsejada.
- `Superseded`: reemplazada por otro ADR.

## Convencion

Los archivos usan el formato `ADR-NNNN-short-title.md`. Un ADR aceptado no se reescribe para cambiar la decision: se crea otro ADR que lo reemplaza.
