# Contributing

Gracias por interesarte en contribuir a PIA.

## Reglas generales

- Mantener el enfoque modular y empresarial del proyecto.
- No introducir lógica de negocio en el sprint 1.
- Respetar la estructura propuesta y la arquitectura existente.
- Mantener documentación actualizada cuando se modifique el diseño o la organización del repositorio.

## Conventional Commits

Usamos mensajes de commit con el siguiente formato:

- feat: para nuevas funcionalidades.
- fix: para correcciones.
- docs: para cambios en documentación.
- chore: para tareas de mantenimiento y estructura.
- refactor: para mejoras internas sin cambiar comportamiento.
- test: para agregar o modificar pruebas.

Ejemplo:

```text
feat: add initial API foundation structure
```

## Proceso

1. Crear una rama desde main.
2. Implementar cambios con foco en claridad y mantenibilidad.
3. Asegurar que la documentación y los archivos relevantes estén actualizados.
4. Abrir un pull request describiendo el cambio.
